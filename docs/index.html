What is Kanso?

Future of web-apps

Modern web development increasingly means pushing more logic onto the client. This provides us with fast and responsive interfaces, but not without cost. When grafted onto traditional web-frameworks, rich interfaces require the complete re-implementation of features on both the server and client-side. Form validation, Templating and URL routing must either be handled at both ends or become fragile and insecure.

What's needed is a harmonious environment, which can deliver enhanced interfaces without breaking the web. Search engines and other users still need access to key content, and all this needs to be delivered in a way that is fast and scalable.

CouchApps and CouchDB provide us with the building blocks to make this vision a reality. A JavaScript environment for querying and storing your data, accessible over a RESTful API from the web-browser, which can scale down to your mobile phone and up to your cluster in the cloud.

CouchApps made simple

The problem with CouchApps, is that although using the same language on the server and the client, the environment is different. This leads people to implement client-side only solutions, using whole new APIs and JavaScript frameworks such as Sammy or Backbone.

This is where Kanso comes in, an attempt to harmonize the environment between CouchDB and the browser by implementing the design doc API client-side. On top of this foundation will be built one of the first real attempts at a powerful, full-featured CouchApp framework. Designed with non-JavaScript fall-backs and logic sharing in mind.
